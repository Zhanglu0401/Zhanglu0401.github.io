一、电脑使用ADB命令提取boot.img文件

1.1 ADB 介绍

adb的全称为Android Debug Bridge，就是起到调试桥的作用。通过adb我们可以在Eclipse中方便通过DDMS来调试Android程序，说白了就是debug工具。

提取流程

首先确保我们的设备是Root后的一个状态才可以进行boot的提取

步骤：

  • 首先我们将手机进入开发者模式，将我们的USB调试打开

  • 将电脑上的adb命令窗口打开

  • 输入 adb devices 检查设备是否连接

  • 进入shell模式

  • 我们需要申请root权限，否则可能会因为文件不可读写，而导致无法提取boot内核

    • 输入 su 获取权限

  • 我们首先要查找内核分区所在位置，先查找快捷方式，我们进入/dev/block目录下by-name文件夹：

    • cd /dev/block/by-name

  • 输入ls 查找当前文件夹列表找到 boot 文件或者AB分区后的boot_a 或 boot_b

  • 查找boot真实地址 输入 ls -l boot文件名

  • 最后，因为我们boot地址为/dev/block/sde11，所以我们使用以下 dd 命令提取保存即可：

    • dd if=/dev/block/sde11 of=/sdcard/boot.img

如果我们在转出boot时出现 /dev/block/sde11: Permission denied 这代表adb没有获取到手机root权限

​

我们只需要在magisk中超级用户给shell获取权限，就可以重新执行- dd if=/dev/block/sde11 of=/sdcard/boot.img进行提取了，这时候我们打开手机存储/sdcard/boot.img就可以看到我们的boot.img包了
